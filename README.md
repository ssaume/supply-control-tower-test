# 供需承諾控制塔｜GitHub Pages 互動原型

這是一個不需要後端、不需要 npm、不需要框架的單頁互動原型。將 `index.html`、`styles.css`、`app.js` 放在 GitHub repository 根目錄，即可透過 GitHub Pages 發布。

## 原型範圍

### 全球工廠

- 台灣：4座工廠
- 中國：6座工廠
- 泰國：8座工廠
- 合計：18座工廠

### 產品、料號與BOM

原型包含21種產品：UPS不斷電系統、嵌入式電源供應器、變壓器、路由器、散熱風扇模組、直流無刷馬達、車載充電器、直流電能轉換器、牽引逆變器、電動車驅動馬達、馬達控制器、直流／交流電動車充電樁設備、變頻器、PLC控制器、LED照明模組、太陽能逆變器、儲能系統（ESS）、微電網控制器、新風系統、安控攝影機、投影機。

- 每種產品建立3～6個成品料號
- 合計93個成品料號
- 每個料號具多階BOM
- BOM最深不超過5階

### 需求事件

原型預建108筆需求日期事件，分布於18座工廠。

- 同一來源批次若有不同需求日期，會拆成不同需求事件。
- 每筆事件均保留來源批次、需求日期、來源、狀態、客戶或內部用途、產品、料號及數量。
- 需求來源包含：客戶預測、內部預測、客戶PO、計畫庫存、緊急詢單。
- 狀態包含：待整合、待確認、已確認、已轉內部SO、已取消。
- 已確認事件可在畫面上轉為內部SO，轉換後立即出現在出貨總覽。

### 出貨與內部SO

初始示範資料包含30筆內部SO；操作需求總覽後可繼續新增。

- 僅列出已轉成內部SO的需求事件。
- 依出貨工廠所在地與目的地，自動判定內銷或外銷。
- 顯示內部SO、關聯需求事件、工廠、客戶、目的地、產品、料號、數量、需求日期、計畫出貨日、Incoterm與出貨狀態。
- 出貨狀態包含：待排程、備料中、待出貨、已出貨、延遲風險。

### 供應網路

原型建立310條供應關係，涵蓋：

- 外部供應商購料或現成模組
- 友廠供應半成品／功能模組
- 友廠代工SMT
- 跨廠成品或半成品備援調撥

供應網路可依產品類型、成品料號及供應關係篩選，並查看週需求、可供量、覆蓋率、前置時間與風險狀態。

## 主要功能

1. **總覽控制塔**：需求日期事件、內部SO、需求變更風險與決策事項。
2. **需求總覽**：BY廠區列出依需求日期拆分的需求事件，並支援狀態推進及轉內部SO。
3. **出貨總覽**：列出已成為內部SO的需求事件，區分內銷、外銷及出貨狀態。
4. **需求變更事件**：需求版本、來源、成品料號、數量、日期、優先順序與凍結區差異。
5. **有限／無限模擬**：調整增量、交期與優先級，查看可達量和供需缺口。
6. **BY產品／BY客戶分析**：以相同需求快照切換兩個維度比較有限與無限結果。
7. **供應網路與BOM**：全球供應關係、供給覆蓋及多階BOM展開。
8. **瓶頸分析**：產能、物料、友廠SMT、跨廠模組與人力限制。
9. **責任人Commit**：加班、友廠代工、跨廠調料、替代料等處理方案。
10. **跨部門決策與追蹤**：比較方案、核准新版本並追蹤承諾差異。

## 本機使用

直接用瀏覽器開啟 `index.html` 即可。也可以在資料夾內啟動簡易伺服器：

```bash
python -m http.server 8000
```

再開啟：

```text
http://localhost:8000
```

## 部署到 GitHub Pages

1. 在 GitHub 建立新的 repository。
2. 將 `index.html`、`styles.css`、`app.js` 上傳至 repository 根目錄。
3. 進入 repository 的 `Settings` → `Pages`。
4. 在 `Build and deployment` 選擇 `Deploy from a branch`。
5. Branch選擇 `main`，Folder選擇 `/ (root)`。
6. 儲存並等待GitHub完成發布。

## 示範資料說明

資料由前端JavaScript產生，沒有連接真實ERP、APS、MRP或物流系統。事件狀態、內部SO、Commit與決策會儲存在瀏覽器 `localStorage`；按左下角「重設示範資料」即可恢復初始狀態。

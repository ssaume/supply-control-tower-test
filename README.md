# 供需承諾控制塔｜GitHub Pages 互動原型

這是一個不需要後端、不需要 npm、不需要框架的單頁互動原型。將 `index.html`、`styles.css`、`app.js` 放在 GitHub repository 根目錄，即可透過 GitHub Pages 發布。

## 原型範圍

### 全球工廠

- 台灣：4座工廠
- 中國：6座工廠
- 泰國：8座工廠
- 合計：18座工廠

每座工廠都有主要生產成品，並依產品技術相似性形成產品聚落，例如電源系統、車用電力電子、電動車動力、工業自動化、能源與照明、熱管理與馬達、資通訊與影像。

### 產品與料號

原型包含21種產品：

UPS不斷電系統、嵌入式電源供應器、變壓器、路由器、散熱風扇模組、直流無刷馬達、車載充電器、直流電能轉換器、牽引逆變器、電動車驅動馬達、馬達控制器、直流/交流電動車充電樁設備、變頻器、PLC控制器、LED照明模組、太陽能逆變器、儲能系統（ESS）、微電網控制器、新風系統、安控攝影機、投影機。

- 每種產品自動建立3～6個成品料號
- 合計93個成品料號
- 每個成品料號有多階BOM
- BOM最深不超過5階

### 供應網路

原型建立326條供應關係，涵蓋：

- 外部供應商購料或現成模組
- 友廠供應半成品／功能模組
- 友廠代工SMT
- 跨廠成品或半成品備援調撥

供應網路畫面可依產品類型、成品料號、供應關係篩選，並查看週需求、可供量、覆蓋率、前置時間和風險狀態。

## 主要功能

1. **總覽控制塔**：全球、地區、工廠風險與產品範圍。
2. **需求變更事件**：需求版本、成品料號、數量、日期、優先順序與凍結區差異。
3. **有限／無限模擬**：調整增量、交期與優先級，查看可達量和供需缺口。
4. **BY產品／BY客戶分析**：以相同需求快照切換兩個維度比較有限與無限結果。
5. **供應網路與BOM**：全球供應關係圖、供給覆蓋、多階BOM展開。
6. **瓶頸分析**：產能、物料、友廠SMT、跨廠模組等限制。
7. **責任人Commit**：加班、友廠代工、跨廠調料、替代料等處理方案。
8. **跨部門決策室**：比較成本、交付、跨廠協同及其他客戶影響。
9. **執行追蹤與Highlight**：承諾與實際差異、原因歸屬及升級處置。
10. **情境導覽**：由需求變更一路導覽到執行追蹤。

## 本機使用

直接用瀏覽器開啟 `index.html` 即可。若瀏覽器對本機檔案有限制，也可以在資料夾內啟動簡易伺服器：

```bash
python -m http.server 8000
```

再開啟：

```text
http://localhost:8000
```

## 部署到 GitHub Pages

1. 在 GitHub 建立新的 repository。
2. 將以下三個檔案上傳至 repository 根目錄：
   - `index.html`
   - `styles.css`
   - `app.js`
3. 進入 repository 的 `Settings`。
4. 選擇左側 `Pages`。
5. 在 `Build and deployment` 選擇 `Deploy from a branch`。
6. Branch 選擇 `main`，Folder 選擇 `/ (root)`。
7. 儲存後等待 GitHub 建立網站網址。

## 示範資料說明

資料由前端JavaScript在瀏覽器中產生，沒有連接真實ERP、APS、MRP或供應商資料。使用者的Commit、決策與事件狀態會儲存在瀏覽器 `localStorage`；按左下角「重設示範資料」可以恢復初始狀態。

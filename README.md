# Supply-chain Control Tower — MRP16

Build `2026.07.15-MRP16`

此版本修正 GitHub Pages 部署後，手工需求保存與審批流程可能觸發「網頁沒有回應」的問題。

主要調整：

- 需求保存後不再立即展開舊需求的工單／採購明細。
- 先讓瀏覽器完成畫面更新，再啟動多階 MRP。
- 產能計算改用週期公式與例外行事曆差額，不再逐日建立大量行事曆物件。
- MRP 完成後只更新目前頁面，不再同步重算產區現況、模擬、瓶頸與所有其他模組。
- 計畫工單、採購單與 MRP 執行紀錄改在瀏覽器空閒時分段保存。
- 接單 LT 核准、退回、升級與跨部門方案核准改為局部保存、局部重繪。
- 接單 LT 核准後才非同步重算 MRP，並防止重複點擊。

部署時請將 ZIP 解壓後的檔案直接放到 GitHub Pages 發布根目錄。

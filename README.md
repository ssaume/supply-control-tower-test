# 供需承諾控制塔｜IMPORT8

Build：`2026.07.14-IMPORT8`

這是可直接部署到 GitHub Pages 的純前端互動原型，不需要 Node.js、npm、資料庫或後端服務。

## 本版重點

### 1. 空白需求環境

- 初始需求事件：0筆
- 初始需求變更事件：0筆
- 舊版瀏覽器 localStorage 使用不同版本鍵值，不會自動載入先前示範需求
- 左側「清除需求資料」可再次清空所有使用者建立的需求與需求變更事件

### 2. 新增「需求管理／需求匯入」

需求匯入支援兩種方式：

1. CSV／JSON批次檔案上傳
2. 人工單筆KEY單

人工KEY單流程（不需自由輸入文字）：

1. 選擇需求來源
2. 從成品料號下拉選單選擇SKU
3. 從下拉選單選擇可生產工廠
4. 從清除前示範資料的候選值選擇數量與需求日期
5. 提出者、部門、客戶／用途、目的地與需求說明均使用來源連動下拉選單
6. 保存後立即出現在需求總覽

可使用包內的 `demand-import-template.csv` 測試檔案匯入。

### 3. 需求來源關係鏈

正常需求鏈：

- 客戶預測 → 客戶PO
- 內部預測 → 內部需求 → 計畫庫存
- 計畫庫存 → 客戶PO消耗

詢單獨立鏈：

- 緊急詢單 → 客戶PO

緊急詢單不再與正常預測／補庫流程混在同一條線上，需求總覽關聯圖會獨立顯示詢單鏈。

### 4. 每筆需求的供給耗用關係

需求總覽新增「供給覆蓋」欄位，顯示：

- 既有成品庫存
- 生產中工單
- 待新增／調整生產
- 既有供給覆蓋率

展開需求事件後，會看到標準化資訊磚：

- 庫存倉別與批號
- 生產中工單編號、品項、分配數量與進度
- 尚未被庫存或在製覆蓋的需求缺口
- 後續內製、委外工單及採購PO

### 5. 響應式與文字檢查

資訊磚已調整：

- 取消重要文字單行省略
- 長工單編號、BOM節點、工廠與供應商名稱可自動換行
- 資訊磚欄位在不同解析度自動由5欄縮為3欄、2欄或1欄
- 手機版表單、工廠選項與供給分配磚改為單欄

實測寬度：

- 桌面：1366px
- 手機：390px

標準化資訊磚與欄位未出現水平溢出。

## CSV欄位

| 欄位 | 必填 | 說明 |
|---|---|---|
| source | 是 | 客戶預測、內部預測、內部需求、計畫庫存、緊急詢單、客戶PO |
| sku | 是 | 成品料號 |
| factory | 是 | 可生產該成品的工廠代碼 |
| qty | 是 | 需求數量，正整數 |
| demandDate | 是 | YYYY-MM-DD |
| proposer | 建議 | 提出者 |
| proposerDept | 建議 | 提出部門 |
| audience | 否 | 客戶名稱或內部用途 |
| destination | 客戶PO建議 | 國家／城市 |
| note | 否 | 備註 |

JSON可使用相同欄位名稱，根節點可為陣列，或使用 `{ "events": [...] }`。

## GitHub Pages部署

1. 解壓縮部署包。
2. 將包內檔案直接放在GitHub Pages發布branch的根目錄。
3. 不要只上傳ZIP，也不要在發布根目錄外再多包一層資料夾。
4. GitHub repository進入 `Settings → Pages`。
5. 選擇 `Deploy from a branch`，指定發布branch與 `/ (root)`。
6. 部署完成後強制重新整理瀏覽器。

成功載入時，左側下方會顯示：

`Build 2026.07.14-IMPORT8`

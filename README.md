# 供需承諾控制塔｜GitHub Pages 互動原型

這是一個不需要後端、不需要 npm、不需要框架的單頁互動原型。將 `index.html`、版本化 CSS 與 JavaScript 檔放在 GitHub repository 根目錄，即可透過 GitHub Pages 發布。

## 原型範圍

### 全球工廠

- 台灣：4座工廠
- 中國：6座工廠
- 泰國：8座工廠
- 合計：18座工廠

### 產品、料號與BOM

原型包含21種產品：UPS不斷電系統、嵌入式電源供應器、變壓器、路由器、散熱風扇模組、直流無刷馬達、車載充電器、直流電能轉換器、牽引逆變器、電動車驅動馬達、馬達控制器、直流／交流電動車充電樁設備、變頻器、PLC控制器、LED照明模組、太陽能逆變器、儲能系統（ESS）、微電網控制器、新風系統、安控攝影機、投影機。

- 每種產品建立3～6個成品料號
- 合計93個成品料號
- 每個料號具多階BOM
- BOM最深不超過5階

### 需求事件與關係鏈

原型預建175筆需求日期事件，分布於18座工廠。每個需求日期都是獨立事件，並透過上游、下游事件編號保存轉換關係。

需求來源與轉換關係重新定義為：

- 客戶預測 → 客戶PO
- 緊急詢單 → 客戶PO
- 內部預測 → 內部需求 → 計畫庫存
- 內部預測 → 內部需求 → 計畫庫存 → 客戶PO（客戶PO消耗既有計畫庫存）

其中客戶PO可能來自「預測完整轉換」、「緊急詢單轉單」或「計畫庫存消耗」。計畫庫存會顯示原始數量、已消耗數量與剩餘數量。

- 需求來源：客戶預測、緊急詢單、內部預測、內部需求、計畫庫存、客戶PO。
- 狀態：待整合、待確認、已確認、已轉換、部分消耗、已被PO消耗、已轉內部SO、已取消。
- 非客戶PO事件只顯示規劃用途，不具備正式客戶、出貨目的地、內外銷與物流資訊。
- 只有客戶PO可以轉為內部SO。

### 出貨、內部SO與運輸

初始示範資料包含14筆由客戶PO轉成的內部SO；使用者可以在需求總覽繼續將其他客戶PO轉成SO。

- 出貨總覽只列出「來源為客戶PO」且「已轉內部SO」的事件。
- 依出貨工廠所在地與客戶目的地判定內銷或外銷。
- 顯示客戶PO來源方式、客戶、目的地、產品、料號、數量、需求日期、計畫出貨日、運輸模式、Incoterm與出貨狀態。
- 運輸模式包含：陸運、海運、空運、快遞、鐵路、複合運輸，可獨立篩選。
- 計畫出貨日依運輸模式的示範前置時間反推。
- 出貨狀態包含：待排程、備料中、待出貨、已出貨、延遲風險。

### 供應網路

原型建立310條供應關係，涵蓋：

- 外部供應商購料或現成模組
- 友廠供應半成品／功能模組
- 友廠代工SMT
- 跨廠成品或半成品備援調撥

供應網路可依產品類型、成品料號及供應關係篩選，並查看週需求、可供量、覆蓋率、前置時間與風險狀態。

## 主要功能

1. **總覽控制塔**：需求日期事件、內部SO、需求變更風險與決策事項。
2. **需求總覽**：BY廠區列出日期事件、提出者、提出日期、版本更換次數及來源關係鏈；支援客戶預測／詢單轉PO、內部預測轉內部需求、內部需求轉計畫庫存、計畫庫存被PO消耗，以及客戶PO轉內部SO。
3. **出貨總覽**：只列出客戶PO形成的內部SO，區分內銷、外銷、運輸模式及出貨狀態；點擊出貨列可回查對應工單與採購進度。
4. **需求變更事件**：直接由需求總覽的上游事件與客戶PO衍生，顯示事件編號、來源批次、來源鏈、需求日期、數量、狀態、客戶與目的地。
5. **有限／無限模擬**：調整增量、交期與優先級，查看可達量和供需缺口。
6. **BY產品／BY客戶分析**：以相同需求快照切換兩個維度比較有限與無限結果。
7. **供應網路與BOM**：全球供應關係、供給覆蓋及多階BOM展開。
8. **生產供應途程**：依成品的製造先後呈現來料齊套、SMT、模組製造、成品總裝、測試與包裝；每站顯示承接工單、執行工廠及投入前必須齊套的採購PO或下階工單。
9. **產區現況**：比較各廠平均Loading、超載與吃緊資源日，並用資源×日期時間網格顯示每天生產的產品、料號、數量、工時與負荷率。
10. **瓶頸分析**：產能、物料、友廠SMT、跨廠模組與人力限制。
11. **責任人Commit**：加班、友廠代工、跨廠調料、替代料等處理方案。
12. **跨部門決策與追蹤**：比較方案、核准新版本並追蹤承諾差異。

### 需求履行、工單與採購追溯

需求總覽與出貨總覽的資料列均可點擊展開。展開區會先合併所有內製工單、委外工單與採購PO，按照需求完成日／需求到料日由早到晚排列，讓使用者由上往下掌握履行時程。

展開資訊提供兩種顯示模式：

- **資訊磚**：以橫向標準化資訊磚顯示類型、狀態、BOM、責任單位、數量、需求時間與進度。
- **清單列表**：使用相同資料與排序，以表格方式進行大量單據比較。

點擊任一工單或採購PO後，系統會根據BOM父子關係突出所選單據、上階消耗工單及同一分支的下階工單／採購PO，其他不相關單據會淡出。工單與採購PO都可作為追溯起點。

展開資訊已統一使用標準化資訊規格：

- 摘要資訊磚固定顯示生產平均進度、物料到料率、內製工單、委外工單與採購PO數量。
- 工單資訊磚固定顯示工單類型、狀態、BOM階層、工單編號、生產項目、生產單位、數量、開始日、完成日、生產進度，以及該階工單物料由庫存或採購PO供給的數量。
- 採購資訊磚固定顯示狀態、BOM階層、採購PO、採購品項、供應來源、總需求量、庫存撥料量、PO訂購量、已到量、承諾到料日與到料進度。
- BOM下鑽中的目前工單與直接下階工單／採購PO，也沿用同一套資訊磚規格。

- 事件層先列出所有對應生產工單與物料採購PO。
- 生產工單明確標示「內製工單」或「委外工單」，委外範圍包含友廠模組供應與友廠SMT代工。
- 工單顯示BOM階層、生產項目、生產工廠／協力廠、數量、計畫期間、完成進度與風險狀態。
- 採購清單顯示採購PO、BOM節點、供應商、訂購量、已到料量、承諾日、到料率與催交風險。
- 點擊任一工單後，下方會依該工單所在的BOM節點，只展開直接下一階的內製工單、委外工單與採購PO。
- 可沿「成品組裝 → 功率／控制模組 → PCBA → 電子料」逐階下鑽，最深支援五階BOM。
- 客戶PO尚未轉SO時顯示規劃／待釋放狀態；轉成SO後，進度會搭配備料、待出貨、已出貨或延遲風險呈現。


### 生產供應途程與供應網路視角差異

在需求或出貨事件的展開區點擊「生產供應途程」，會切換至該成品的獨立途程頁面。

- **供應網路與BOM**著重「從哪裡來、由哪些零件與模組組成」，適合查看全球供應來源、BOM階層、供給覆蓋率與跨廠關係。
- **生產供應途程**著重「先做什麼、在哪裡做、每一道站點何時要齊料」，適合評估製造順序、站點交接、委外執行與物料齊套風險。

途程預設包含六個標準階段：來料檢驗與齊套、SMT／PCBA、核心模組製造、成品總裝、功能測試與校正、包裝與出貨準備。各階段會顯示：

- 執行工廠與產能資源
- 承接的內製／委外工單
- 投入前必須齊套的採購PO或下階工單
- 計畫期間、完成進度與齊套風險

### 需求變更事件與需求總覽同步

需求變更事件不再使用獨立的事件假資料，而是由需求總覽中的客戶PO與其上游事件動態建立：

- 客戶預測 → 客戶PO
- 緊急詢單 → 客戶PO
- 內部預測 → 內部需求 → 計畫庫存 → 客戶PO

事件明細可回查上游需求事件、客戶PO事件、來源批次、關係鏈、需求日期、數量、狀態與計畫庫存消耗結果。使用者在需求總覽新建立客戶PO時，系統也會同步新增需求變更事件。

### 產區現況與產能時間網格

- 支援全球、地區或單一工廠範圍。
- 可選擇7、14或21天時間範圍。
- 可篩選SMT／PCBA、模組製造、組裝、測試／校正資源。
- 上方比較各廠平均Loading、超載資源日、吃緊資源日與計畫工時。
- 下方時間網格以「產能資源 × 日期」呈現每日主要生產內容。
- 每個格子顯示產品、成品料號、計畫數量及Loading；點擊可查看工時、需求事件與來源。
- 另顯示待排能力、預防保養與換線準備。

## 本機使用

直接用瀏覽器開啟 `index.html` 即可。也可以在資料夾內啟動簡易伺服器：

```bash
python -m http.server 8000
```

再開啟：

```text
http://localhost:8000
```

## 部署到 GitHub Pages

1. 在 GitHub 建立新的 repository。
2. 將 `index.html`、`styles-v20260714-demand6.css`、`app-v20260714-demand6.js` 與 `.nojekyll` 上傳至 repository 根目錄。
3. 進入 repository 的 `Settings` → `Pages`。
4. 在 `Build and deployment` 選擇 `Deploy from a branch`。
5. Branch選擇 `main`，Folder選擇 `/ (root)`。
6. 儲存並等待GitHub完成發布。

## 示範資料說明

資料由前端JavaScript產生，沒有連接真實ERP、APS、MRP或物流系統。事件狀態、內部SO、Commit與決策會儲存在瀏覽器 `localStorage`；按左下角「重設示範資料」即可恢復初始狀態。

### 生產供應途程：水平網路圖

「生產供應途程」提供兩種呈現方式：

- **途程卡片**：由上往下查看每一道製程的工單、地點、資源與齊套供給。
- **網路圖**：由左至右呈現完整途程，左側為來料與齊套起點，右側為包裝與出貨終點；各途程下方掛載內製工單、友廠／委外工單及採購 PO。

網路圖支援水平捲動，適合在桌面上查看跨廠生產順序，也能在手機上左右滑動。

## 可收合階層式功能導覽

左側功能列目前支援兩種層級的收合：

1. **整體功能列收合**：桌面版可按品牌區右側的 `«`，將側欄縮成只顯示功能編號的窄版；再按 `»` 即可展開。主內容會同步向左延伸，不會留下空白。
2. **功能群組收合**：功能依用途分成兩組，群組可各自展開或收合：
   - **總覽與分析**：總覽控制塔、需求總覽、出貨總覽、有限／無限模擬、供應網路與 BOM、生產供應途程、產區現況、瓶頸分析。
   - **事件審批**：需求變更事件、責任人 Commit、跨部門決策室、執行追蹤與 Highlight。

側欄與群組的收合狀態會保存在瀏覽器中。透過頁面快捷連結或情境導覽切換到某個功能時，系統會自動展開該功能所在群組。手機版仍使用滑出式完整功能列，兩個功能群組也可獨立收合。


## 生產供應途程：站點生態圖

生產供應途程目前提供三種視角：

- **途程卡片**：適合逐站閱讀工單、供給、資源、日期與完成進度。
- **途程網路圖**：以六段製程為主軸，由左側途程起點連到右側途程終點。
- **站點生態圖**：參考供應鏈生態網路的分欄呈現方式，從左到右分為供應來源、前段製造、模組與總裝、測試與配送、需求終點。

站點生態圖會呈現：

- 外部採購、現成模組、友廠與委外製造節點
- 各生產途程的執行工廠、產能資源、期間與完成進度
- 物料／下階工單供給至哪一道製造站點
- 途程之間的前後依賴關係
- 客戶、目的地、需求數量及需求日期
- 正常、進度偏低與齊套風險狀態

點擊任一節點後，該節點及直接上下游會保留強調，其餘節點與關聯線會淡出。站點生態圖支援水平捲動，手機版可左右滑動。

本版本也放大生產供應途程的說明、標籤、工單、採購供給與網路節點文字，避免關鍵資訊使用過小字級。

## DEMAND6 更新內容

### 需求來源轉換關聯圖

需求總覽新增可互動的來源轉換圖，使用四欄視角呈現「需求訊號、內部確認、庫存準備、正式訂單」。每個節點顯示目前範圍內的需求事件數與需求量，連線下方顯示實際轉換筆數。點擊節點可直接篩選該來源。

### 事件提出與版本治理

每一筆需求事件均包含：

- 提出者
- 提出部門
- 提出日期（日顆粒度）
- 最後改版日期
- 目前版本
- 累積更換版本次數

使用者在原型中確認、轉換或轉SO時，版本資訊會同步更新。

### 工單物料耗用來源

每一階工單會彙總其BOM分支中的物料需求，分別標示：

- 庫存撥料數量
- 採購PO數量
- 使用庫存的品項數
- 使用採購PO的品項數

採購PO資訊也會拆出總物料需求、庫存撥料與實際需下單數量。

### 生產供應途程視圖切換

「途程卡片、途程網路圖、站點生態圖」三個按鈕已使用相同高度、寬度、字級與選取樣式；桌面版等寬排列，手機版改為垂直排列。
